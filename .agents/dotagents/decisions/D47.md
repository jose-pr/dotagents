# D47: root AGENTS.md is the committed public library-interface doc; private working knowledge moves under .agents/

Date: 2026-07-20

## Decision

A repo's root `AGENTS.md` is now a COMMITTED, package-shipped, agent-facing
description of how to USE the library — public API/exports each with its
signature/accepted args/defaults/required, return-or-contract, key env vars, and
gotchas, header-file-style — so a consuming agent understands the library without
reading its source. Private agent working knowledge (architecture notes, gotchas,
per-directory guidance) lives in `<project>/.agents/AGENTS.md` (repo-root or subtree,
deeper wins). `.gitignore` excludes `.agents/`, `CLAUDE*`, `.claude` — but NOT root
`AGENTS.md`. Both `README.md` and root `AGENTS.md` must ship inside the built package
(sdist + wheel).

Supersedes the prior "two files, two owners" model, where root `AGENTS.md` was the
agent-maintained *private* notes file (gitignored) and `.agents/AGENTS.md` the
user-managed directives. Working notes now consolidate under `.agents/AGENTS.md`
(agent-written, user-editable, user wins on conflict). The global `~/.agents/AGENTS.md`
is unaffected — it remains the global directives, neither a repo library doc nor
repo working notes.

## Rationale

Consuming agents kept having to parse a dependency's entire source to learn its public
API. A committed, packaged interface doc — like a header file — lets an agent use a
library directly. The old "gitignore every AGENTS.md" rule made that impossible, and
putting the private notes at root `AGENTS.md` collided with the public-doc name. Source
of change: user request 2026-07-20 (mid the duho `AGENT_HELP` / agent-consumable-docs
thread).

## Changes

- `src/dotagents/_overlay/AGENTS.md`: leakage rule drops `AGENTS.md` from the ignore
  list; a "two kinds of AGENTS.md" block replaces "two files, two owners".
- `overlays/flows/flows/REPO.md`: meta-files gains the root-`AGENTS.md` library-doc
  standard and requires README + AGENTS.md to ship in the package; gitignore line updated.
- `overlays/flows/flows/EXEC.md`: read step + Collateral point working notes at
  `.agents/AGENTS.md`; a public-API change also updates root `AGENTS.md`.
- `overlays/references/references/.gitignore`: no longer excludes `AGENTS.md`.
- `overlays/python/kb/PYTHON.md`: force-include `README.md` + `AGENTS.md` in the wheel.
- `overlays/references/references/master_refactoring_plan.md`: working-notes refs point
  at `.agents/AGENTS.md`.

## Migration (per repo, "for now")

Move a repo's existing root `AGENTS.md` private notes into `.agents/AGENTS.md`, free
root `AGENTS.md` for the public library doc, and update `.gitignore` to stop excluding
`AGENTS.md`. Not retroactive across all repos in one pass; apply as each repo is touched.

## Refinement (2026-07-20, same day, user)

The committed public docs split BY LOCATION into three roles (not one interface doc):
- repo-root `AGENTS.md` = a dev-facing project OVERVIEW (what it is, code layout, entry
  points, how to build/develop/use at a high level; links to the module headers);
- `src/**/AGENTS.md` = header-file-style per-module public-API docs, colocated with the
  code (one per source module/package dir) — the "header files";
- `.agents/AGENTS.md` = the private working notes (unchanged).
Packaging: `src/**/AGENTS.md` ship in the wheel naturally (inside the package tree);
force-include only the repo-root `AGENTS.md` + `README.md`. Core `_overlay/AGENTS.md`
and `flows/REPO.md` reflect this three-role model.
