---
name: D43
description: link/sync never adopt or copy back a <project>/.agents that is itself a git checkout, so a hosted-runner session-source clone can't be swallowed into the private store
type: decision
---

Observed (2026-07-20, hosted web session): a session that lists the private agents
repo itself as a *source* gets it cloned by the harness to `<project>/.agents` — on a
session branch, with the origin pointing at the runner's in-session git proxy
(`127.0.0.1:<port>`). The SessionStart bootstrap runs `dotagents link <project>`
concurrently, and D37's first-link adoption did exactly what it was designed to do to
a plain directory: moved the *entire checkout* — `.git` included — into
`~/.agents/projects/<name>/`, then symlinked. Result: a full nested repo (foreign
proxy remote, session branch) sitting untracked inside the private repo. The next
`dotagents sync`'s `git add -A` would have recorded it as a bare gitlink and pushed
that to the private repo; `sync`'s copy-mode copy-back had the same swallow for a
real-dir project (worse: `overwrite=True`).

Decision: a `.agents` whose `.git` exists (dir, or file — worktrees/submodules) is
presumed managed by something else and is **never** adopted or copied back.

- `link_project`: log a `skip:` line, leave the checkout in place untouched, and skip
  linking entirely (the project simply uses the checkout as its `.agents`, which in
  the observed case *is* the agents repo — CLAUDE/AGENTS context still loads).
  `--force` keeps an escape hatch consistent with the existing conflict path: back the
  checkout up to `.agents.bak*` (git state intact) and link the store.
- `sync_agents`: log `skip copy-back:` and proceed with the normal git sync of the
  agents repo itself.

Not chosen: adopting content while excluding `.git`. The observed colliding checkout
is the whole agents repo — its tree (global config, every project's store) is wrong
for a per-project store even without `.git`, and guessing which subset to adopt is
exactly the kind of silent magic the conflict path avoids. CI covers guard, skip
messages, non-adoption, and the `--force` backup path.
