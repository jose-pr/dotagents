The monolithic `payload/` (one full config `install` copied 1:1) is replaced by:
- **base overlay** `src/dotagents/_overlay/` — the neutral minimum `init` lays down
  (managed-block `AGENTS.md`/`CLAUDE.md`, `README.md`, `dotagents/DECISIONS.md`). Stays
  package data so `init` works with no repo. **No empty scaffold dirs/`.gitkeep`** —
  `dotagents/decisions/`, `findings/` are created lazily.
- **opt-in overlays** `overlays/<name>/` — composable example bundles, each installing
  to the same relative path in the dest, each with an `overlay.toml`
  (`name`/`description`/`requires`/`routing`). Split by concern:
  `flows` (PLAN/EXEC/REVIEW/REPO + MODELS), `recovery` (kb/RECOVERY), `tools`
  (summarize_run + compare_bench, the user-overlay helpers), `references` (neutral
  templates), `python`/`node`/`rust` (kb + manifests + CI), `agents` (antigravity).
- **required tooling** at top-level `tools/` (`audit_config.py`, `leak_check.py`) —
  dotagents' own validators, not a user overlay (the `tools_as_dotagents_subcommands.md` draft plan). Bundled into the
  pyz as `_tools`; `dotagents audit` finds it there or in a repo checkout.

`payload/` and `payload/examples/` are gone entirely. The commit-gate audit now runs
`--root .` against the repo (manifest remapped to the new paths). Supersedes [[D24]]'s
"payload isolated under payload/" layout. Overlays are **examples, not required, and
the source is pluggable** — see [[D36]] for the installer's (deliberately minimal)
overlay handling.
