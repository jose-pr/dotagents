---
name: D40
description: dotagents sync authenticates the private repo itself (token + proxy-rewrite bypass), so a standalone CLI sync works without being run through the private-sync Stop hook
type: decision
---

Symptom: a direct `dotagents sync` against `~/.agents` on a hosted runner failed —
`git pull` reported `could not read Password for 'http://local_proxy@127.0.0.1:PORT'`
and `git push` returned HTTP 403 — even though the same repo synced fine at session
Stop. Root cause: the auth+bypass lived only in `overlays/private-sync/hooks/_agents-git-auth.sh`,
which the Stop hook sources before calling the CLI. The CLI's own `_git()` (in `_link.py`)
ran plain `git`, so a direct run inherited the global `url.<proxy>.insteadOf https://github.com/`
rewrite and hit the scoped in-session proxy, which won't serve an out-of-scope private
repo (and a repo whose name starts with `.` can never be added to session scope — see
D37 / CLOUD_ENV). So the CLI worked only as a hook-driven subprocess, never standalone.

Fix — port the hook's logic into the CLI so `dotagents sync` is self-sufficient
(`_agents_git_auth()` in `_link.py`, applied to the pull/push git calls):

- **No token** → no-op; ordinary git (unchanged behavior for non-private-sync users).
- **Token, no github→proxy rewrite** (local machine) → a per-command `-c` credential
  helper (`credential.https://github.com.helper` reads `$DOTAGENTS_AGENTS_TOKEN` at auth
  time) with `useHttpPath=false`. No global-config mutation — preferred over the shell
  hook's `git config --global`, since the CLI shouldn't leave state behind.
- **Token + rewrite active** (hosted runner) → an isolated `GIT_CONFIG_GLOBAL`
  (`GIT_CONFIG_SYSTEM=/dev/null`) built in a temp file with the credential helper, the
  effective `user.name`/`user.email` (so a rebase/commit still has a committer), and the
  egress-proxy CA bundle. Dropping the normal global config removes the `insteadOf`
  rewrite, so `origin`'s stored tokenless `https://github.com/...` URL resolves directly.
  The temp file is unlinked in a `finally`.

Scope: auth is applied only to the network ops (pull/push); `add`/`status`/`commit` stay
plain (local, and identity is preserved in the isolated config anyway). The credential
snippet mirrors `_agents-git-auth.sh` verbatim, so hook and CLI auth stay identical; the
Stop hook keeps sourcing the shell helper (belt-and-suspenders, and it still exports the
bypass for any other git it runs). Token is never written to `.git/config`.

Verified live in the pkgforge web container: standalone `dotagents sync` logged
`bypassing it for token auth`, pulled, and pushed at exit 0 where the stock 0.2.0 CLI had
403'd. `audit_config.py --root .` PASS. This is also why `kb/CLOUD_ENV.md`'s earlier note
that `dotagents sync`'s push 403s through the proxy no longer holds once this ships.
