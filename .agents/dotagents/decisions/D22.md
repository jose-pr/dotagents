Changelogs, release notes, README/docs, source comments, tests, workflows, benchmarks describe user-visible behavior / bug rationale / mechanism directly. No private plan filenames, `.agents/plans` paths, phase numbers, reviewer IDs, workflow history. Provenance goes in the private log, not committed files. Mechanically checked by `audit_config.py --repo-hygiene`; see [[D24]].
