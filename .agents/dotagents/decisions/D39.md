---
name: D39
description: cloud-setup self-heals the container-start clone (retry/backoff + a persisted SessionStart recovery hook), so one early egress-race failure can't permanently disable the environment
type: decision
---

Symptom (seen across fresh sessions): `~/.agents` never appeared, the `dotagents` CLI
was never installed, and no hooks were wired — the whole private-sync system silently
did nothing. Root cause: `tools/cloud-setup.sh` runs at container start, *before* the
in-session egress/proxy is reliably up, and its step-2 clone `exit 0`'d on the first
failure. Because hook wiring is the LAST step, that early exit skipped it — so the
SessionStart hook (`private-sync-start.sh`, which itself re-clones when `~/.agents` is
missing) was never registered, and nothing could recover. One early clone failure
disabled the environment for good; my `ls-remote` with the same token/auth succeeded once
egress was up, confirming the race (not a credentials/permission problem).

Fix — two layers so the last step always gets a chance to run:

- **Retry with backoff** around the clone (5 attempts, 1/4/9/16s; a partial target is
  removed before each retry). Handles the common case where egress comes up within
  ~30s of container start.
- **Persisted recovery hook** when retries are still exhausted: copy this self-contained
  script to `~/.dotagents/cloud-setup.sh` (outside `~/.agents`, which doesn't exist yet)
  and wire a SessionStart hook `sh "$HOME/.dotagents/cloud-setup.sh"` into
  `~/.claude/settings.json`. The next session starts with egress up, re-runs the whole
  bootstrap, and succeeds. The recovery script is a copy of the bootstrap (not the
  private-sync hook) precisely because those hooks live *inside* the repo that failed to
  clone — the recovery path must not depend on the clone it's recovering.

Symmetry: the settings edit is one idempotent helper (`_dg_settings present|absent
[snippet]`). A failed run adds the recovery hook; the first successful run merges the
private-sync snippet AND removes the recovery hook, so steady state is exactly the
private-sync hooks with no leftover. `$HOME` is expanded by the harness at hook-run time
(stored literally), matching the existing `"$HOME/.agents/..."` hook style. `cloud-setup.sh`
stays in the audit's EXIST_ONLY set (not content-scanned); the new paths introduce no
personal/machine markers.
