name: D37
description: private-agents git sync — ~/.agents is one private repo; per-project .agents symlink to projects/<name>; dotagents link/sync + a private-sync overlay
type: decision
---

Goal (user steer): keep the per-user config **and** every project's private `.agents`
(plans, kb, findings) in **one private git repo**, synced across machines and cloud,
without committing any of it into the (often public) project repos.

Design:
- **The global `~/.agents` IS the private repo.** Root = per-user config (already the
  install target); a new `projects/<name>/` tree holds each project's private `.agents`
  payload. Nothing new to reconcile — `init`/`install` still target `~/.agents`.
- **Per-project link = symlink** `<project>/.agents -> ~/.agents/projects/<name>`. The
  Leakage rule (base overlay) already `.gitignore`s `.agents/` in every project, so the
  symlink never lands in the public repo; all private state lives in / syncs through the
  one private repo. Builds on D17–D20 (per-project `.agents`, plans always under the
  project) — this centralizes those dirs instead of scattering them per machine.
- **Naming = directory basename** (user choice), so a local `~/code/app` and a cloud
  `/home/user/app` converge on `projects/app`; `--name` overrides on basename collisions.
  Git-remote-slug keying was considered (more robust across renames/forks) but rejected
  as heavier than the user wanted.
- **`--copy` fallback** mirrors the store as a real dir for no-symlink environments
  (Windows without privilege); a symlink failure auto-falls back. Copy mode is two-way
  (sync-back before refresh) so symlink stays the recommended default.
- **Adoption on first link**: an existing real `.agents/` moves into an empty store,
  then the symlink replaces it — so `dotagents link` works in a project that already has
  `.agents/`. A genuine conflict (both sides carry real content) needs `--force`.

Surface:
- `dotagents link [PATH] [--name --agents-dir --copy --force --dry-run]` and
  `dotagents sync [--project --name --remote --no-pull --no-push -m --dry-run]`, backed
  by `src/dotagents/_link.py` (pure stdlib — no `pathlib_next`, works under a plain `pip
  install`). `sync --remote` bootstraps a fresh repo (`git init` + `origin`) in one call;
  `sync --project` copies a copy-mode project back into its store before committing.
- `overlays/private-sync/` ships `kb/PRIVATE_SYNC.md` (routing: "set up / understand
  private-agents git sync") + cloud `hooks/private-sync-{start,stop}.sh` (SessionStart
  clone/pull + link; Stop sync-back) + `settings.snippet.json` for `~/.claude/
  settings.json`. Cloud auth comes from a `DOTAGENTS_AGENTS_REMOTE` env secret, never a
  committed file — the private repo's scope is sensitive.

Boundary: creating the actual private GitHub repo is the user's outward-facing action;
the CLI makes it turnkey (`sync --remote`) but does not create/push a remote repo on its
own. The public `jose-pr/dotagents` repo is unaffected — this is tooling for a *separate*
private `.agents` repo. Relates to the deferred `dotagents overlays` subcommand (D36):
`private-sync` is a normal opt-in overlay applied by path today.
