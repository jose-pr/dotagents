Date: 2026-07-21

## Decision

A repo has **no root `AGENTS.md`**. The two jobs D47 gave it are already carried
by files that do them better:

| Job | Home |
| --- | --- |
| Public, agent-consumable interface | `src/**/AGENTS.md` — one header per module/package dir, committed + package-shipped |
| Private working knowledge (architecture, gotchas, per-dir guidance) | `<project>/.agents/AGENTS.md` — gitignored, symlinked into the private store |
| Human-facing overview | `README.md` — already exists, already shipped |

Supersedes [D47](D47.md), which made root `AGENTS.md` a committed public
library-interface doc, and the refinement in `78457a7` that recast it as a
dev-facing overview pointing at the module headers.

## Why

D47 was written when the module headers did not exist yet, so root `AGENTS.md`
was the only place a public interface doc could live. That is no longer true —
every repo now carries real `src/**/AGENTS.md` headers (`pydhcp` has four, one
per subpackage). With the headers in place, root `AGENTS.md` became a middle
layer that:

- **duplicates `README.md`** for the "what is this, how is it laid out" job, and
  the two drift;
- **duplicates the module headers** for the API job, at lower fidelity, because a
  root-level file cannot stay current with each module the way a colocated header
  does (D47's own "same commit" requirement is much easier to honor next to the
  code);
- **collides by name** with the private `.agents/AGENTS.md` and the global
  `~/.agents/AGENTS.md`, which is the confusion D47 was itself trying to fix.

Three files named `AGENTS.md` with three meanings was the problem. Two — one
colocated with code, one under `.agents/` — is learnable.

## Consequences

- `.gitignore` may go back to excluding `.agents` (slashless) without the D47
  carve-out for a root `AGENTS.md`. `leak_check.py`'s carve-out comment and the
  `--repo-hygiene` expectations follow.
- The `<AGENT>.md → @AGENTS.md` pointer convention is unaffected: it applies to
  the **global** `~/.agents/AGENTS.md`, not to repo roots.
- Existing root `AGENTS.md` files are not deleted blind — their content is
  triaged first: interface material moves into the relevant `src/**/AGENTS.md`,
  private/architecture material into `.agents/AGENTS.md`, human-facing material
  into `README.md`.
