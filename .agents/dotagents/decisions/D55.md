Date: 2026-07-21

## Decision

The `dotagents` repo itself is **excluded from `dotagents link`**. Every other
project links its `.agents` into `~/.agents/projects/<name>/`; this one keeps a
real, tracked `.agents/` directory.

## Why

`dotagents` is the one repo that intentionally **tracks** `.agents/` — 69 files
of sanitized design log (`DECISIONS.md`, `decisions/D*.md`, `NOTES.md`) and
public plans. That is the public-repo exception recorded in its own
`.agents/AGENTS.md`, and it is the reason its `.gitignore` deliberately does
**not** carry a `.agents` line.

`link_project` replaces `<project>/.agents` with a symlink to the store. Run
here it would:

1. adopt the 69 tracked files into `~/.agents/projects/dotagents/`, then
2. `shutil.rmtree` the real directory and symlink in its place — which git sees
   as **deleting all 69 tracked files** and adding one untracked symlink.

The design log would leave the public repo. `link --dry-run` surfaces the
mismatch as a warning ("`.gitignore` does not exclude `.agents`"), but the
warning reads as advice to add the line — which would be exactly the wrong fix
here.

## Consequence

- The private store `~/.agents/projects/dotagents/` is never created.
- Work *on this repo* is committed to this repo, like any other tracked content.
- Anything genuinely private about developing dotagents belongs in
  `~/.agents/dotagents/findings/` (the global private area), not in a store.
- A future `link` should refuse (not just warn) when `<project>/.agents` has
  tracked files: the adopt-then-rmtree path is destructive to repo content, and
  the git-checkout guard from [D43](D43.md) does not catch it — a tracked plain
  directory has no `.git` of its own. Tracked via a draft plan.
