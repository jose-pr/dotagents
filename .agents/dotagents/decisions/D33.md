New `src/dotagents/` package; `install.py` becomes a thin shim over `dotagents.cli.main()`. Built on the author's `duho` (declarative CLI, pinned 0.1.1) + `pathlib_next` (copy/sync/URI, pinned 0.8.0). Subdecisions:
- **init** writes a minimal neutral base overlay (`src/dotagents/_overlay/`) — no flows/model-routing, imposes nothing. **install** writes the full opinionated `payload/` ([[D24]] behavior).
- **--from <uri-or-path>** on both, resolved via `pathlib_next` `UriPath` (lazy import; `pip install "dotagents[uri]"` hint if the extra is missing). A plain pip-install (no repo) has no bundled full payload → `install` requires `--from`; payload stays single-sourced at repo root, no `src/` copy that could drift.
- **build-pyz** vendors duho/pathlib_next via `pip install --target` (pinned, no extras — pyz core is stdlib-only), then `zipapp`. PyPI package declares them as ordinary deps; vendoring is pyz-only.
- **--bin-dir** writes `dotagents` (sh) + `dotagents.cmd` wrappers, warns with the literal PATH-export line if not on PATH.
- **init merge (never clobber)**: `AGENTS.md`/`CLAUDE.md` content is delimited by `<!-- dotagents:begin -->`/`<!-- dotagents:end -->` markers — create if missing, prepend block if present without markers (user content untouched, scaffolding first), refresh only between markers if present (idempotent). Detection by marker, never prose. `CLAUDE.md`: skip if `@AGENTS.md` already present. `--force` = full replace-with-backup.
Full plan: `.agents/plans/dotagents_cli_package.md`. Gotchas live in repo-root `AGENTS.md` arch-notes. See [[D33-log-split]] (same session's reorg).
