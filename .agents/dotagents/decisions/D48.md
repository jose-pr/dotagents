Date: 2026-07-20

## Decision

The rule *never print or echo `DOTAGENTS_*` environment values* lives in the
**always-on** Leakage rule in the core `AGENTS.md`, not only in
`kb/CLOUD_ENV.md`.

## Why

`DOTAGENTS_AGENTS_TOKEN` was printed into a session transcript **twice**
(2026-07-19 and 2026-07-20), both times by a first-look `env` / `printenv`
triage while diagnosing the cloud bootstrap. Both required an owner PAT
rotation (both notes now in `dotagents/findings/processed/`).

The guard existed after the first incident — but it was written into
`kb/CLOUD_ENV.md`, which is **load-on-demand**. The routing line only fires
once the agent knows it is debugging the cloud environment, and the leaking
`env` call is precisely how an agent *finds that out*. An on-demand rule cannot
protect against the command that precedes the decision to load it.

Generalizes: a rule whose violation happens during *orientation* must be
in-context before orientation starts. On-demand routing is for rules whose
trigger the agent can recognize first.

## Cost

+~140 bytes on the always-on core, which is already over its 2500B budget
(see [D49](D49.md)). Accepted: the alternative is a third rotation.
