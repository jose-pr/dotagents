Date: 2026-07-20

## Decision

`audit_config.py`'s `BUDGETS` overruns print `WARN:` and do **not** fail the
run. Keep it that way — a hard failure would block unrelated work on a
judgement call — but treat every WARN as recorded debt: the current sizes are
written down here, and a change that grows an over-budget file states why.

## Current state (2026-07-20, after the origin/main merge)

| File | Size | Budget | Over by |
| --- | --- | --- | --- |
| `src/dotagents/_overlay/AGENTS.md` | 3649B | 2500B | +1149B |
| `overlays/flows/flows/REPO.md` | 7012B | 3000B | +4012B |

`AGENTS.md` grew +215B in this merge: the [D48](D48.md) secret guard (~140B)
and the slashless-`.agents` clarification the merge forced (see
[D47](D47.md) vs `1c9bf7c`). Both are load-bearing.

## Why this needed recording

The WARN was invisible: `audit_config.py --root .` prints `PASS` on the last
line, and the size table scrolls above it, so a session that checks the exit
code or the final line sees a clean run. The overruns predate this merge and
had not been noticed.

`.agents/dotagents/NOTES.md` still lists the core `AGENTS.md` at 2743B — stale
by 906B. Anyone consulting the moved-rule mapping before re-expanding prose (as
the repo `AGENTS.md` requires) would have been reading a number that stopped
being true several commits ago.

## Compression pass (2026-07-21)

Done, enabled by [D54](D54.md) dropping the root `AGENTS.md` role — the
three-role block was the single largest item in the core.

| File | Before | After | Budget |
| --- | --- | --- | --- |
| `src/dotagents/_overlay/AGENTS.md` | 3649B | **2474B** | 2500B — under |
| `overlays/flows/flows/REPO.md` | 7012B | 6052B | 3000B — still over |

Moved-rule mapping (core `AGENTS.md`, no rule lost):

| Removed prose | Where the rule now lives |
| --- | --- |
| Root-`AGENTS.md` role + its gitignore carve-out | Deleted outright — D54 removes the role |
| "three roles, by location" framing | Two-kind list; the third kind no longer exists |
| `.agents/{kb,flows,references}` full path repetition | Kept once, relative |
| Plans: "descriptive"/"preserve sub-tree"/"creating if needed" | Condensed; the rule is unchanged |
| Load-on-demand preamble verbosity | Condensed; routing semantics unchanged |

**`REPO.md`'s budget is knowingly not met.** It covers layout + CI + six meta
files + versioning, where PLAN/EXEC/REVIEW each cover one topic and all sit
2419–2693B. Cutting 3000B more would delete rules, not prose. The right fix is
splitting it (e.g. meta-files → its own routed file), which is a structural
change, not a compression pass. Left in the debt plan; the budget entry should
be re-based or the file split, not silently trimmed.
